# Evidence pack — `diagnostic-settings-missing` lab

This directory carries the canonical 2026-06-22 Phase A evidence cohort for the `diagnostic-settings-missing` lab plus the derived Phase B gate outputs emitted by `labs/diagnostic-settings-missing/verify.sh`. The claim ceiling is deliberately narrow: this pack proves only what this single `koreacentral` reproduction can support about environment-level `appLogsConfiguration` routing for `ContainerAppConsoleLogs_CL` and `ContainerAppSystemLogs_CL`.

## Capture timeline

Three carry-over framings make the evidence pack readable:

1. **Silent baseline.** The baseline is not merely "0 rows after traffic"; it is the stronger state where the environment reports `destination: null`, both KQL queries classify as `silent_valid_baseline`, and the raw baseline captures show `BadArgumentError: SEM0100 'where' operator: Failed to resolve table` because the `_CL` tables never materialized.
2. **Fix restores ingestion.** The environment-level fix is a single `az containerapp env update --logs-destination log-analytics --logs-workspace-id --logs-workspace-key` operation. The app image, ingress path, workspace target, request pattern, and KQL query text are held constant; only the environment routing state changes before the post-fix KQL is re-run.
3. **Single-variable falsification is bounded, not absolute.** The falsification claim is limited to the intentional env-config payload plus the forced revision lineage used to emit fresh platform events. It does not claim image-digest identity, pod reuse, row-per-request correspondence, or a universal 5-minute ingestion SLA.

Capture phases:

- **Phase A — Live reproduction** (`2026-06-22`): `trigger.sh` captured the null-destination baseline and `fix-and-capture.sh` applied the environment fix, forced a new revision, and captured the populated post-fix state. The Phase A experiment window anchors on `03-curl-before.json.utc_completed = 2026-06-22T09:44:18Z` and `08-curl-after.json.utc_completed = 2026-06-22T09:53:02Z` for a 524-second end-to-end span.
- **Phase B — Offline evidence verification** (`runtime overlay date: generated by verify.sh at execution time`): `verify.sh` is a pure file processor that reads the committed Phase A cohort and emits `14-cohort-integrity-gate.json`, `15-baseline-silent-gate.json`, `16-post-fix-populated-gate.json`, and `17-single-variable-falsification-gate.json`.

## Gate overview

- **Gate 14 — `14-cohort-integrity-gate.json`**: checks (a) canonical-file presence with Strong exact-20 and Fallback ≥18-plus-required-inputs paths, (b) monotonic baseline-to-post-fix temporal coherence with Strong ≤30-minute and Fallback ≤90-minute windows, (c) no unexpected non-junk extras, and (d) this README's literal cross-reference to all four gate filenames.
- **Gate 15 — `15-baseline-silent-gate.json`**: checks (a) `destination == None AND logAnalyticsConfiguration == None`, (b) `int(console_rows) == 0 AND int(system_rows) == 0`, (c) both classifications equal `silent_valid_baseline`, and (d) baseline traffic was actually generated with `requests_ok >= 8`.
- **Gate 16 — `16-post-fix-populated-gate.json`**: checks (a) `destination == "log-analytics"` plus workspace-identity cross-check, (b) `int(console_rows) >= 1 AND int(system_rows) >= 1`, (c) both classifications equal `populated_table`, and (d) the post-fix traffic holder is the single running revision with `trafficWeight == 100` and 3-way revision-name equality across `07`, `08`, and `09`.
- **Gate 17 — `17-single-variable-falsification-gate.json`**: checks (a) the shared env-config surface is exactly `{destination, logAnalyticsConfiguration}` and that whole surface changed together, (b) baseline revision remains in lineage while the post-fix traffic holder is a new revision, and (c) every post-fix revision name starts with `containerAppName + "--"`.

## Honest disclosure

- The baseline silence is evidenced in two layers: parsed JSON (`04-kql-before.json`) and raw CLI stderr (`04-kql-before-console-raw.txt`, `04-kql-before-system-raw.txt`).
- The post-fix population is likewise evidenced in two layers: parsed JSON (`09-kql-after.json`) and raw CLI stdout (`09-kql-after-console-raw.txt`, `09-kql-after-system-raw.txt`).
- `09-kql-after.json` stores `console_rows` and `system_rows` as JSON strings, not numbers. Phase B casts them with `int(...)` before evaluating thresholds.
- `07-revisions-after.json` is a top-level array, not an object wrapper. Phase B treats the file itself as the revision record set.
- The file name `00-verify-run.txt` is preserved for schema stability even though the Phase A script was renamed to `fix-and-capture.sh` during the Phase B overlay.
- Gate 14 evaluates both Strong and Fallback paths and records both explicitly; a gate passes only when every sub-gate passes at least one path.
- Exit-code semantics differ by phase: `fix-and-capture.sh` may exit 2 for H2-falsified, while Phase B `verify.sh` exits only 0 (all gates pass) or 1 (any gate fails).
- Region and tooling context are preserved in `12-region.json`, `10-cli-versions.json`, and `11-cli-containerapp-ext.json` so future readers can compare reproductions apples-to-apples.
- The environment workspace customer ID is intentionally redacted from Phase B gate JSON prose/observed-values to avoid treating a real workspace GUID as shippable documentation text.
- **Image byte-identity NOT cohort-evidenced.** The cohort infers no image change from the Phase A env-var-only app update, but it does not capture image digests.
- **Pod reuse NOT claimed.** The forced `FIXAPPLIED` update creates a new revision under `activeRevisionsMode: Single`; the cohort does not capture pod UIDs.
- **5-minute ingestion latency is not an SLA.** Microsoft Learn does not publish a strict `_CL` table-ingestion SLA for Container Apps log routing; the 300-second wait is an observed sufficient window for this single reproduction.
- **Helloworld stdout volume is not a row-per-request guarantee.** This cohort observed 1 console row and 34 system rows after 10 post-fix requests; Gate 16 therefore uses `>= 1 row`, not a one-row-per-request assumption.

## Reproducibility

```bash
cd labs/diagnostic-settings-missing/
bash verify.sh
```

The verifier reads only the committed evidence files in this directory, emits the four gate JSONs, prints per-gate and total sub-gate pass counts, and exits 0 only when all 15 sub-gates pass.
