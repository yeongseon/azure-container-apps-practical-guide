# Evidence directory

Raw experimental output from `trigger.sh`, `h3-falsification.sh`, and `analyze.py`.

This directory is **permanent and committed to the repository** per the user mandate that raw data must be preserved to avoid re-running the experiment.

## File naming

- `consumption-<timestamp>.jsonl` — scale-ladder samples for Consumption profile
- `dedicated-<timestamp>.jsonl` — scale-ladder samples for Dedicated D8 profile (auto-picked active revision)
- `dedicated-rev<NNNN>-<timestamp>.jsonl` — samples from one specific revision (e.g. when auto-pick targets the wrong revision, or for cross-revision boot_id checks)
- `h3-same-replica-<timestamp>.jsonl` — Part A of proxy validation (idempotence)
- `h3-restart-<timestamp>.jsonl` — Part B of proxy validation (new-replica detection)
- `trigger-<timestamp>.log` — orchestration log for one full run
- `verify-<timestamp>.log` — post-deploy `verify.sh` output
- `deploy-<timestamp>.log` — `deploy.sh` output (Bicep deployment + ACR build)
- `final-state-<timestamp>.log` — manual `az` snapshot of environment + revision list before forensic captures
- `<name>-summary.md` — markdown generated by `analyze.py` from the corresponding JSONL
- `<name>-buggy-stdin.{jsonl,log}` — forensic artifacts retained for tooling regressions; do not analyze
- `<name>-broken-now_ms.log` — forensic artifacts retained for tooling regressions; do not analyze

## Editing policy

- Raw `*.jsonl` and `*.log` files are **append-only with one documented exception**: PII (real subscription IDs, tenant IDs, object IDs, employee paths) MUST be masked per AGENTS.md before commit. Masking is content-preserving (zero-GUID `00000000-0000-0000-0000-000000000000`, generic paths) and is the only allowed mutation.
- Generated `*-summary.md` files MAY be regenerated by re-running `analyze.py` against the same JSONL.
- Adding new runs is allowed; deleting old runs is **not** allowed without an issue documenting why.
